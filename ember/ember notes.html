2.1: Templates and the Router

  • Most work will be done in the /app directory
  • ember generate template application //creates hbs template
  • creates a template under: /app/templates/application.hbs
  • always have an {{outlet}} expression in the application template. this is where templates go.
  • then add an index file... app/templates/index.hbs
  • add an order.hbs file
  • need to use a router to get this template to show. a router manages the application state and maps it
  • configure the router by opeing the /app/router.js file
  • Router.map(function() { }); //this is where we define all app endpoints here
  • Additional Helpers: Ember ships with many other Handlebars helpers for logic. debugger, each, if, input, link-to, log, textarea, unless

2.6: Routes

  • Router > Routes > Templates
  • Router talks to Routes
  • Routes talks to Templates
  • Router: Manages state
  • Routes: Collects needed data and renders templates
  • Templates: Defines HTML
  • Auto-generated Routes: app/router.js > this.route('orders');
  • Generating a route
  • CLI: ember generate route
  • CLI: ember generate route myroutetest //created a /app/routes/myroutest.js, /app/templates/myroutest.hbs, also updated the router.js file with this new route name
  • By doing this, it extends the route so you can do something custom with this route
  • {{#each model as |order| }}
  • Order ID: {{order.id}} | {{order.name}}
  • Common Used Route Hooks: activate (triggered when navigation enters the route), deactivate (triggered when navigation leaves the route), model (returns model data to use in the route and template), redirect (sending user to a different route)
  • this.route('order', {path: '/orders/:order_id'}); //creates a single order detail page (":" is a dynamic segment)
  • {{#link-to "order" order}} //"order" = is the route name. order = order object.

3: Models and Services

  • Generate a Service: aka, singletons. these are available throughout your app
  • Services are good for: Centralized logging, User sessions, WebSocket management, data repos
  • CLI: ember generate service
  • Models: represent the underlying data of the application.
  • Defined in app/models
  • Interacting with Models: Properties are read and set using get() and set()
